{% comment %} Stable polling wrapper. The wrapper element itself (#scan-poller) never gets swapped — only the inner #scan-progress-card is replaced. This means a single failed swap or network blip can't kill the polling chain: the wrapper's `every 2s` timer keeps firing as long as the element exists. To stop polling, scan_progress_fragment returns HTTP 286 when the scan reaches a terminal state. htmx recognizes 286 and stops the poll; it still processes the response body, so an out-of-band #scan-result-panel swap in the same response can update the results inline. Initial render only (from scan_detail.html): we include _scan_progress_card.html directly. The polling response uses _scan_progress_card.html alone (no wrapper). {% endcomment %}
